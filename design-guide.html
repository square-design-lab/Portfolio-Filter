Design System

Config UI Design Guide

The visual language for all Squarespace plugin configuration dashboards.
Apply these rules to every new plugin to maintain a consistent, professional experience.

01 — Layout Structure

Three-panel shell

Every plugin config UI uses the same two-zone shell: a fixed sidebar on the left for settings, and a main content area on the right split into preview + code drawer.

  • 1
    Sidebar width is 380px — never narrower. This gives labels and controls enough room to breathe.
  • 2
    Header height is 60px — both the sidebar header and the main topbar. Keeps alignment on the same baseline.
  • 3
    Preview canvas uses a radial-gradient(#E1E1E1 1px, transparent 1px) dotted pattern at 20×20px — Figma-like signal that this is a canvas.
  • 4
    On mobile (<1024px): sidebar becomes full-screen, main panel becomes full-screen, switching via bottom tab bar (68px tall with safe-area padding).

06 — Code Drawer & Syntax Highlighting

Dark code panel

The code output drawer sits at the bottom of the main content area, always visible. It uses a dark theme with syntax highlighting applied via JS regex replacement.

<style>#gridThumbs{opacity:0;}</style>
<link rel="stylesheet" href="plugin.css">
<script>
window.PORTFOLIO_CONFIG = {
    "layout": "top",
    "searchEnabled": true,
    "itemsPerPage": 8,
    "noResultsText": "No results found."
};
</script>
<script src="plugin.js" defer></script>

Syntax token colors

"key" — #E5BF78 amber
"string" — #87C98F green
true — #79B8FF blue
123 — #FF9D7C orange
<tag> — #6B8CAE slate
window — #C678DD purple
Important: Store the raw (un-highlighted) code in a rawCode variable and use that for clipboard copy — never read innerHTML for copying. The #codeOutput element uses innerHTML for display and the variable for copy.
07 — Mobile Tab Navigation

Bottom tab bar (<1024px)

On mobile the entire UI shifts to a three-tab bottom navigation: Settings, Preview, Code.

↑ Full-screen panel content above
Settings
Preview
Code
  • 1
    Tab bar height: 68px plus env(safe-area-inset-bottom) for iOS notch. Always position: fixed; bottom: 0.
  • 2
    Active tab: solid --accent background, white text/icon, 600 weight, 10px border-radius.
  • 3
    Inactive tab: transparent bg, --text-tertiary color, icon opacity 50%.
  • 4
    Copy Code button must appear in the sidebar header on mobile — users are on Settings tab most often.
  • 5
    Sidebar content fills the full viewport minus the tab bar. Header stays fixed; only .sidebar-content scrolls.
08 — Sidebar Section Order

Recommended section sequence

New plugin config UIs should follow this section order. It mirrors the mental model users have: structure first, appearance second, text last.

1
Layout & Structure — main layout, sidebar/grid widths, filter groups, labels. Open by default.
2
Styling & Colors — preset theme, alignment, spacing, border-radius, color pickers.
3
Item Meta / Display — what info shows on each grid card (tags, year, location, excerpt, etc.).
4
Text Labels — all user-visible strings (All, Reset, Search placeholder, etc.).
5
Motion — animation type, speed, stagger. Hide speed + stagger controls when animation is "None".
6
Advanced Behavior — desktop/mobile filter style, multi-select, counts, truncation, pagination, sort. Collapsed by default.

10 — Do / Don't

Quick rules

✓ Do
  • ✓
    Use --accent for all interactive states (focus, checked, active tabs, CTA button).
  • ✓
    Keep section headers 12px/uppercase/0.07em — always match this exactly.
  • ✓
    Default new boolean toggles to off unless the feature is widely used (like animations).
  • ✓
    Show helper text below controls that aren't self-explanatory (11px/#666).
  • ✓
    Hide dependent controls (e.g., sticky spacing only when sticky is on).
  • ✓
    Store raw code in a variable; use innerHTML only for highlighted display.
  • ✓
    Place "Copy Code" CTA in both the desktop topbar and the mobile sidebar header.
✗ Don't
  • ✗
    Don't hardcode colors outside of the :root block — always reference CSS variables.
  • ✗
    Don't use a sidebar narrower than 380px — labels will truncate.
  • ✗
    Don't add more than 6 accordion sections — it overwhelms users.
  • ✗
    Don't use flat gray thumbnails in preview cards — use the 4-color gradient pattern.
  • ✗
    Don't change the font. Inter is the only typeface for UI; JetBrains Mono for code only.
  • ✗
    Don't use a different accent color — keep #4F46E5 across all plugins for brand consistency.
  • ✗
    Don't copy innerHTML for the clipboard — it will include HTML tags in the output.
